# DQN-AOL-GA Smart Shop Scheduler 2.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a CPU-friendly Smart Shop Scheduler 2.0 in the final-report worktree, replacing the tabular Q controller with a checkpointed Double-DQN controller while preserving all existing scheduling and disturbance baselines.

**Architecture:** Keep the current decoder, metrics, disturbance semantics, and Streamlit entry point. Replace the monolithic `algorithms.py` module with an `algorithms` package, add a focused `rl` package for neural control, and route both initial and dynamic scheduling through a service layer. DQN observes a ten-feature continuous population state and selects one of five shared GA search actions.

**Tech Stack:** Python 3.11, NumPy, PyTorch CPU, pandas, Plotly, Streamlit, pytest, Streamlit AppTest.

---

## File Map

**New packages and responsibilities**

- `algorithms/__init__.py`: public algorithm registry and exports.
- `algorithms/common.py`: `AlgorithmResult`, evaluation helpers, population creation, selection, and shared types.
- `algorithms/operators.py`: crossover, swap/insertion mutation, diversity, and bottleneck local search.
- `algorithms/baselines.py`: FIFO and fixed GA.
- `algorithms/tabular.py`: existing SLGA and CP-AOL-SLGA behavior.
- `algorithms/dqn_ga.py`: generation loop controlled by a `DQNAgent`.
- `rl/actions.py`: immutable five-action registry used by tabular and DQN controllers.
- `rl/state_encoder.py`: ten-feature continuous state encoding.
- `rl/replay_buffer.py`: bounded transition memory.
- `rl/dqn_agent.py`: Q networks, Double-DQN updates, epsilon policy, deterministic seeds.
- `rl/checkpoint.py`: atomic model and metadata persistence.
- `services/scheduling.py`: algorithm construction and schedule runs for the UI.
- `services/training.py`: multi-episode CPU training and progress report.
- `training/train_dqn.py`: command-line training entry point.
- `tests/`: focused unit, integration, disturbance, and UI smoke tests.

**Existing files changed**

- Delete `algorithms.py` after its behavior is moved into the package.
- Modify `app.py`, `disturbance.py`, `visualization.py`, `requirements.txt`, and `.gitignore`.
- Preserve `scheduler_core.py`, `metrics.py`, `data_loader.py`, and included data unless a failing correctness test requires a scoped fix.

```python
class QNetwork(torch.nn.Module):
    def __init__(self, state_size: int, action_size: int):
        super().__init__()
        self.layers = torch.nn.Sequential(
            torch.nn.Linear(state_size, 64),
            torch.nn.ReLU(),
            torch.nn.Linear(64, 64),
            torch.nn.ReLU(),
```

```python
from pathlib import Path

from rl.checkpoint import CheckpointMetadata, load_checkpoint, save_checkpoint
from rl.dqn_agent import DQNAgent, DQNConfig


def test_checkpoint_round_trip_preserves_greedy_policy(tmp_path: Path):
    source = DQNAgent(DQNConfig(10, 5, seed=9))
    state = np.linspace(0, 1, 10, dtype=np.float32)
    path = tmp_path / "agent.pt"
    metadata = CheckpointMetadata("dqn-state-v1", "ga-actions-v1", 9, ("FT06",), 2)
    save_checkpoint(path, source, metadata)
    loaded, loaded_metadata = load_checkpoint(path, DQNConfig(10, 5, seed=99))
    assert loaded.select_action(state, explore=False) == source.select_action(state, explore=False)
    assert loaded_metadata == metadata


def test_checkpoint_rejects_incompatible_schema(tmp_path: Path):
    source = DQNAgent(DQNConfig(10, 5, seed=9))
    path = tmp_path / "agent.pt"
    save_checkpoint(path, source, CheckpointMetadata("old-state", "ga-actions-v1", 9, (), 0))
    try:
        load_checkpoint(path, DQNConfig(10, 5, seed=9), expected_state_version="dqn-state-v1")
    except ValueError as exc:
        assert "state schema" in str(exc)
    else:
        raise AssertionError("incompatible checkpoint was accepted")
```

```python
from rl.dqn_agent import DQNAgent, DQNConfig


def test_dqn_reschedules_breakdown_and_emergency_job():
    jobs, metadata = load_dataset("FT06")
    machines = int(metadata["machines"])
    agent = DQNAgent(DQNConfig(10, 5, seed=23))
    initial = run_dqn_ga(jobs, machines, agent, 20, 8, 23, training=False)
    frozen_before_event = {
        (record["job_id"], record["operation_id"]): dict(record)
        for record in initial.schedule
        if record["end"] <= 20
    }
    updated_jobs, result, logs = dynamic_reschedule(
        initial.schedule,
        jobs,
        machines,
        algorithm="DQN-AOL-GA",
        current_time=20,
        breakdown=(1, 20, 15),
        emergency_route=[(0, 4), (2, 5), (1, 3)],
        emergency_time=20,
        random_seed=23,
        fast_mode=True,
        agent=agent,
        training=False,
    )
    assert len(updated_jobs) == len(jobs) + 1
    assert result.makespan >= 20
    assert any("DQN-AOL-GA" in line for line in logs)
    result_by_key = {
        (record["job_id"], record["operation_id"]): record
        for record in result.schedule
    }
    for key, frozen_record in frozen_before_event.items():
        assert result_by_key[key] == frozen_record
```
